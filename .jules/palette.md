## 2025-03-15 - Enhancing Form Accessibility While Preserving Minimalist UI
**Learning:** When designers request minimal forms without visual labels, simply relying on placeholders fails WCAG requirements and breaks screen-reader utility. Utilizing Tailwind's `sr-only` class to visually hide the `<label>` tags allows the interface to maintain its sleek aesthetics without compromising accessibility and form semantics, providing a clean solution to a common UI/UX friction point. Furthermore, providing immediate button feedback upon login dramatically improves perceived performance over blocking synchronous operations.
**Action:** Always implement `sr-only` explicit `<label>` elements linked via `htmlFor`/`id` and appropriate `autoComplete` attributes on minimalist input fields. Ensure async forms block double submission with a clear, immediate visual loading state inside the submit button.
