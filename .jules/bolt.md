## 2024-03-21 - Extract fast-updating state to prevent massive re-renders
**Learning:** The `DigitalBinder` component is massive (over 60,000 bytes) and contains a `time` state updated every second via `setInterval` for a live clock in the header. Because this state lives at the very top level of `DigitalBinder`, the *entire page component* (including `RoadmapTracker`, all sections, and large lists of text) is forced to re-render every single second. This is a severe performance bottleneck causing constant React reconciliation over thousands of DOM nodes.
**Action:** Extract fast-updating, localized UI elements (like live clocks) into their own isolated, small React components. This isolates the state updates and limits the re-render scope to just the tiny component itself, preserving the live-update functionality while preventing the entire parent page from unnecessarily re-rendering 60 times a minute.
