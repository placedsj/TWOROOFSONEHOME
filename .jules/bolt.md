## 2024-03-20 - Unnecessary periodic re-renders in static roadmap tracker
**Learning:** `RoadmapTracker.tsx` was using `setInterval` to trigger a re-render every 60 seconds by updating a `now` state (`const [now, setNow] = useState(new Date())`), even though this state wasn't directly used in the component's output. The actual calculations (`calculateSunriseProgress`, etc.) internally instantiated `new Date()` directly, making the `now` state purely a forced re-render mechanism. Since the milestone calculators measure progress over months and years, a per-minute re-render is completely unnecessary and wastes CPU cycles, especially when left open in a background tab.
**Action:** Remove the `setInterval` and the unused `now` state from `RoadmapTracker.tsx`. Components rendering long-term milestones (months/years) do not need minute-by-minute forced re-renders. If periodic updates are required, they should be isolated to a dedicated child component (like a clock) to avoid massive parent component tree re-renders.
