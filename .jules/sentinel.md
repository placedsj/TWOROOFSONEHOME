## 2024-03-07 - Add missing `autoComplete` attributes and explicit `<label>` elements to login form inputs
**Vulnerability:** The login form inputs in `pages/Login.tsx` were missing explicit `<label>` elements linked via `htmlFor`/`id` and lacked `autoComplete` attributes. While not a direct exploit vector like XSS or SQLi, this degrades security by preventing password managers and browser autofill from functioning correctly and securely filling credentials, encouraging poor password practices (like typing or reusing weak passwords). It also violates the security and accessibility coding standard for the project.
**Learning:** Security features like browser built-in credential management require explicit semantic HTML (`id`, `name`, `autoComplete`, and explicitly linked `<label>` elements) to function correctly. This is an easy oversight when rapidly prototyping UI using placeholder inputs.
**Prevention:** Enforce semantic HTML standards during code review. Ensure all interactive form inputs always include explicitly linked labels and appropriate `autoComplete` attributes as part of a baseline security/accessibility checklist.
