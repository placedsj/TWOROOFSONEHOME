## 2024-05-24 - Client-side Secret Exposure via Vite config
**Vulnerability:** The `vite.config.ts` explicitly maps sensitive server-side environment variables (`env.GEMINI_API_KEY`) to client-side globals like `process.env.API_KEY` and `process.env.GEMINI_API_KEY`.
**Learning:** Vite's `define` configuration performs static string replacement at build time. Injecting API keys this way embeds the plaintext secret into the client bundle, directly exposing it. Furthermore, any remediation must ensure `process.env` is polyfilled (`define: { 'process.env': {} }`) to prevent the application or third-party libraries from crashing with a `ReferenceError: process is not defined`.
**Prevention:** Never use Vite's `define` to map sensitive server-side environment variables to the client. Keep secrets on the server and use proxies or server-side functions for sensitive API calls. If client-side access is absolutely required and intended, use public variables (e.g., `VITE_PUBLIC_...`) but be aware they will be visible. In this case, removing the mapping is critical.
