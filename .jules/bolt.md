## 2024-05-24 - High-Frequency State Updates in Large Parent Components
**Learning:** Found a performance bottleneck where `pages/DigitalBinder.tsx` contained a `setInterval` triggering state updates every second for a clock display. Because `DigitalBinder.tsx` is a massive component containing many deeply nested sub-components, this meant that *the entire UI hierarchy* was unnecessarily re-rendering 60 times a minute. This codebase-specific pattern (putting small, frequently updating data at the top-level route) forces React to do unnecessary reconciliation work across a massive DOM structure.
**Action:** When adding small UI features that update frequently (timers, counters, live status), always isolate them into their own leaf components (e.g. `LiveClock`). State that updates every second should be pushed down as low in the component tree as possible to prevent parent re-renders.
