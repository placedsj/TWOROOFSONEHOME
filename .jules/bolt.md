## 2026-03-07 - Component Isolation Strategy\n**Learning:** The `pages/DigitalBinder.tsx` component was unnecessarily re-rendering every second because it held the `time` state updated via `setInterval`. Since `DigitalBinder` is a large component with many children, this caused significant rendering overhead.\n**Action:** Isolate high-frequency state updates (like timers) into dedicated leaf components (e.g., `LiveClock`). This ensures that only the small UI element actually changing (the clock display) re-renders, protecting the parent and sibling components from unnecessary render cycles.
