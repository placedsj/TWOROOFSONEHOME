## 2024-05-14 - Semantic Form Context and Non-Blocking UI Additions
**Learning:** Legacy prototype implementations of interactive forms (such as `pages/Login.tsx`) frequently rely on blocking elements like `alert` or omit accessibility essentials like `<label>` bindings and `autoComplete` tags. When these `<form>` wrappers exist without explicitly typed components to interpret them, screen readers and password managers fail to parse the interaction context correctly.
**Action:** Always add semantic structural components, including `id`/`htmlFor` pairings (even if the `label` is visually hidden with `.sr-only`), and supply `autoComplete` attributes mapped to the field purpose. Replace any blocking prototype UI with realistic visual states (like `isLoggingIn` + `lucide-react` `Loader2`) alongside `setTimeout` to mimic native interaction pacing without blocking the main thread.
